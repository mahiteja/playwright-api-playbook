# 📘 Playwright API Testing Playbook

Live site: 🔗 [https://mahiteja.github.io/playwright-api-playbook/](https://mahiteja.github.io/playwright-api-playbook/)

This project is a practical reference guide for using Playwright to test APIs and API-driven workflows. It is organized as a browsable playbook with examples, patterns, and decision points you can apply in real test suites.

## ✅ What It Covers

- Core API testing capabilities such as request setup, HTTP methods, headers, query params, payloads, assertions, and request chaining.
- Authentication approaches including API keys, basic auth, bearer tokens, OAuth flows, cookie-based sessions, NTLM, mTLS, and persisted auth state.
- Advanced Playwright usage such as reusable fixtures, retries, parallel execution, environment variables, CI/CD, reporting, schema validation, soft assertions, performance checks, snapshots, and custom matchers.
- Special topics like GraphQL, WebSocket testing, server-sent events, HAR tracing, redirect handling, HTTP protocol checks, and Playwright features added in newer versions.
- Guidance on when Playwright is the right choice compared with other API automation tools.

## 👥 Target Audience

- QA engineers who want a clearer Playwright API testing reference.
- Test automation engineers building or improving API and end-to-end test suites.
- Developers who need concise examples for validating backend services through Playwright.
- Teams evaluating Playwright for API testing in CI/CD pipelines.

## 🎯 Purpose

The goal of this playbook is to help readers quickly find the right pattern for a given API testing problem and move from concept to implementation with less trial and error.
